Types of components

A component is defined as a standalone object which can be applied to a page.

Pathway components

Pathways are the primary method of in-page navigation. They provide a link to another page or website as a tile within the context of the page.

Choosing a pathway component

There are a number of pathway components defined by difference in appearance. These differences enable a hierarchy to be built into the actions on a page. The4 choice pathway component shouold be considered within the context of the page.

Displaying multiple pathways as a list

Components that include 'list' in the name display a grid of multipe pathways. Each tile is added as an item in the one component. The display of each item is treated the same across the entire list, eg. if you choose to hide the description it will be hidden across every component.

Content components

These components format in-page copy and imagery to present the content as clearly as possible.

Menu components

These global components enable the user to navigate in an intuitive and consistent manner across the site. These include global and page navigation.

Anatomy of components

Element Data source* Notes
Label Multiple The short label displayed able the title within the page header and pathways
Title Page This is the page title displayed in the header and pulled into pathways.
Long description Page This is the long description which is displayed in the header and pulled into search result cards.
Short description Page This short description is added at a page level and displayed as the description within pathways to that page.
Headline Component Header styled content that is set into an individual component.
Body copy Component Any body copy set into an individual component. Includes text formatting for heading, paragraph, list and link styles.
Link indicator / icon Component This is the small icon which appears at the bottom left-hand corner of a pathway.
Container Component This is the background style for the container.

* Data source refers to where the component pulls the element from. See below for an explanation of the different data sources.

Data source glossary of terms

Data source Description
Page The content is pulled from the current page meta data.
Target page The content is pulled from the page which the component is linking to.
Component The content is set within the individual component.
External import The content is managed by an external source, eg. Stenning.
Multiple The content can come from more than one source, eg. it is pulled from the target page by default but can be overridden in the component settings.
