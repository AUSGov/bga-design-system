Guidelines for use

Colour use

Colour use in infographics should be limited to one colour set. Colours from different sets should not be combined. Each colour set has one primary colour and one secondary colour. The primary colour should be the main colour used in the graphic, with the secondary colours providing contrast and highlights.

The extended colour set is only for us in complex graphs and should be avoided when creating infographics.

Font use and text colour

Infographics and graphs should use Google OpenSans. Primary font weights are:

  • OpenSans-Semibold
  • OpenSans-Regular

Other weights can be used when particular elements of the design require emphasis but should be used sparingly.

The default font colour is black #333333. Accessible colours from the chosen colour set can also be used for large text in infographics.

Visual elements

All visual elements in infographics and graphs are 2-dimensional, not 3-dimensional.

Visual elements in infographics can be broken into two categories:

  • BGA style illustrations
    • Illustration style follows the guidelines outlined in the illustrations section of the style guide.
    • Colour rules for illustrations follow those outlined above and therefore differ slightly from those outlined in the illustrations section of the style guide.
  • Icons
    • Where possible use icons from the BGA icon set.
    • If the BGA icon set does not contain a required icon then any new icons created should match the icon style.
    • Google material design provides free icons that match the BGA icon set. To download Google material icons navigate to https://material.io/resources/icons/?style=baseline and select the “Filled” theme.

Graphs

Graphs should always have data labels and should never rely solely on colour or contrast to convey information.

Colours in graphs should be limited to one colour set. Graphs with complex data may use the extended graph colour set instead.

Online Accessibility

When infographics and graphs are used online they need to meet accessibility requirements. This can include:

  • Using HTML to display numbers and text wherever possible, rather than incorporating them into an image file.
  • Using labels with graphs to convey information rather than relying on colour and contrast.
  • Using ARIA roles recommended in the WAI-ARIA Graphics Module (see https://www.w3.org/TR/graphics-aria-1.0/#roles)
  • Using the <longdesc> HTML attribute to provide a detailed explanation of the infographic content or including a link to the content description in a collapsible section of the page.
